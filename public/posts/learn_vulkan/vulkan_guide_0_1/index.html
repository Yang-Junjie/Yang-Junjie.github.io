Chapter 1 : Initialization and render Loop#

Vulkan Initialization#

与 OpenGL 不同的是 Vulkan 中几乎所有的东西都需要自己手动管理，所以我们使用 VkBootstrap 库来简化这种繁琐的工作
Vulkan 的初始化工作顺序一般是：初始化VkInstance -> 初始化VkPhysicalDevice -> 初始化VkDevice -> 初始化VkSwapChain

VkInstance#

VkInstance 是 Vulkan 的入口，它代表了Vulkan Api 的上下文，在创建 VkInstance 的时候，可以指定扩展，还可以开启验证层，验证层是一种能帮程序员检测错误的程序，通常一个应用中只有一个全局VkInstance.

VkPhysicalDevice#

VkPhysicalDevice 用于查询你电脑中的GPU信息，比如有哪些GPU，每个GPU的属性，这个GPU支不支持这个功能。并且可以或者这个GPU的句柄

VkDevice#

VkDevice 是我们通过程序与GPU通信的核心，我们想与GPU进行通信必须经过它。

SwapChain#

SwapChain 不是Vulkan的核心功能，因为Vulkan最主要还是解决GPU的问题，而SwapChain是用于将GPU渲染的图像显示到屏幕上的，对于不同的操作系统有不同的窗口管理，所以这不是Vulkan规范中的功能，但是Vulkan给出了一种名为扩展的功能，能让程序员针对不同的情况（不用的操作系统）编写专门的代码，而我们想将内容渲染到窗口上就需要使用VK_KHR_surface这个扩展，当然如果你不想将内容渲染到窗口上就不需要使用这个扩展，你可以将内容渲染到一张纹理中然后保存到图片，或者你只想使用vulkan与gpu通信，使用gpu的Compute shader等等。 SwapChain的尺寸对应了你的窗口的尺寸，如果窗口的大小变化那么你应该重新创建swapchain
值得注意的一点是不同的平台swapchain所需要的图片格式不同，所以在呈现的时候必须询问平台的格式，再创建swapchain图片。
Swapchain 维护了一个图像队列。即我们常说的交换缓冲区，我们可以设置不同的交换方式去呈现图片。
